在Markdown中使用缩进

  说实话，Markdown用起来很方便，但是不处理行首的空格以及忽略连续空格确实很不习惯，虽然这锅应该甩给HTML。文章的话还是觉得每段空两格看的更舒服一些，事实上中文习惯也应该是这样吧。交一篇所有段首都没有空格的作文，怕不是要被语文老师训话（笑）。

  既然知道这是HTML的问题，那么其实可以从HTML中寻找解决办法。在这之前，先理解一个概念，HTML字符实体。

在 HTML 中，某些字符是预留的。
在 HTML 中不能使用小于号（<）和大于号（>），这是因为浏览器会误认为它们是标签。
如果希望正确地显示预留字符，我们必须在 HTML 源代码中使用字符实体（character entities）。字符实体类似这样：
&entity_name;
或
&#entity_number;

ViaHTML字符实体

  当然，与之相关的空格或空白符不止以上三种。更多的请参考HTML实体参考手册。

  其实说明很清楚了，使用全角空格两个，正好可以空出两个中文大小的位置。但还是来对比一下。

  普通空格占据的宽度是受字体的影响的，因此用来缩进可能并不合适。要进行与文字大小相关的缩进长度，可以使用多个半角空格或全角空格来实现。
