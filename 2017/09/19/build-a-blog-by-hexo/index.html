使用GitHub搭建博客

hexo

  既然正好用GitHub和Hexo搭建了一个博客，那就把搭建流程简单的介绍一下，详细的一搜一大把，我就不啰嗦了。然后再记录一些需要注意的地方。

Hexo

  Hexo是一个博客框架，使用Markdown解析文章，生成静态网页。GitHub有个Pages的功能，可以用网页展示个人或者项目，其一个限制就是只能使用静态网页。当然，在其他环境使用Hexo也是可以，只不过GitHub免费，不限流量，还可以绑定自己的域名，是一个比较好的选择。

准备

  Hexo是使用Node.js实现的，然后还依赖于GitHub（或其他仓库），所以需要用到以下两个程序。

  • Node.js
  • Git

  Node.js不需要直接使用到，所以不会的话没什么关系。但是Git还是需要会一些基本操作。会Git百利而无一害。Git的教程也很多，这里粘个廖老师的Git教程。另外上面的Git链接不翻墙可能打不开，可以参考这里。建议在部署之前，先熟悉一下Git的使用流程。虽然实际使用可能只需要设置下账号和SSH-Key就行，但还是那句话，只有好处没有坏处。

主题

  玩博客嘛，肯定少不了换皮的操作主题。当然可以自己做主题，做不了的就只能使用大大们分享的主题了，非常感谢大大们的贡献。各式各样的主题可以在这里找到。点击图片会跳转到一个网页，一般就是作者的博客，也可能仅是一个演示页面。点击名字会跳转到GitHub页面，通常来说都有使用方法。一般分为以下3步：

1.git clone 仓库到本地themes文件夹下。
2.修改主题的配置，配置文件是主题目录下的_config.yml。当然也可能不需要修改。
3.修改根目录下_config.yml中theme的值为想要的主题的名字。

多台电脑更新

  使用多台电脑更新是比较常见的需求。使用U盘或网盘之类的备份当然是可以的，但是未免有些麻烦。个人认为比较好的方案就是将整个hexo文件夹也上传到GitHub之类的代码仓库。事实上观察一下.gitignore，作者已经考虑到这种方法，忽略文件都已经帮忙写好了。

  这里也有两个方案，第一个方案就是在博客的仓库新建一个分支用来存储。但是这里有个问题就是历史commit都会有记录，比如删了文章，但是还是能在历史找到。当然也可以重建之类的，但是多些操作终究是有点麻烦。第二个方案就是新建一个私人仓库用来存储。比如Coding就提供有限数量的私人仓库。如果push失败，可能需要删除主题目录下的.git目录。

  另外hexo的部署是可以将不同的部分上传到不同的仓库，比如将public的上传到博客的仓库，而将hexo目录上传到备份仓库。详细的可以参考hexo-deployer-git。

结语

  总得来说Hexo搭建起来还是比较简单的，使用也很容易，而且不用管理服务端，很省事。只是简单的记录一下流程，还有更多的东西有待挖掘。
